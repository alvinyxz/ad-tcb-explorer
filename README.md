# TCB Explorer — Static Companion Site

A small, interactive web app that lets readers explore the topic-cluster-bridge
(TCB) results: 1,674 advertising articles, 72 topics, 8 clusters, 2,556 bridges.

The site is fully client-side. The browser fetches a handful of JSON files from
`data/` and renders everything live; there is no server. Hosting is free on
GitHub Pages, and the URL embeds cleanly inside Google Sites.

## Folder layout

```
99_website/
  index.html            # the entire app (one file)
  data/                 # generated by 01_Script/00_Revision/09_Website.R
    meta.json
    topics.json
    clusters.json
    papers.json
    topic_papers.json
    bridges.json
  README.md             # this file
```

## One-time data export

In R (from the project root):

```r
source("01_Script/00_Revision/09_Website.R")
```

That writes the six JSONs into `02_Output/00_Revision/99_website/data/`.
Re-run any time the underlying analyses change.

## Local preview

`fetch()` will not work from a `file://` URL, so open the site through a
tiny local server. From the `99_website/` folder:

```bash
# Python 3 (already installed on macOS)
python3 -m http.server 8080
# then open http://localhost:8080
```

If anything is missing or malformed, the page shows a clear error pointing at
the JSON file that failed to load.

## Deploying to GitHub Pages

1. Create a new public repo, e.g. `tcb-explorer`.
2. Copy the contents of this `99_website/` folder into the repo root, then
   `git add . && git commit -m "Initial site" && git push`.
3. In the repo on GitHub, go to **Settings → Pages**, set **Source** to
   `Deploy from a branch`, **Branch** to `main` (folder `/root`), Save.
4. After about a minute the site is live at
   `https://<your-username>.github.io/tcb-explorer/`.

To update later: re-run `09_Website.R`, copy the refreshed `data/` folder into
the repo, `git push`. The site updates within a minute.

## Embedding in Google Sites

1. Open the Google Site, click **Insert → Embed → By URL**, paste the GitHub
   Pages URL.
2. Sites will render the live app inside an iframe. Resize the frame in edit
   mode so the tabs and tables fit comfortably.
3. Publish the Site. Visitors interact with the explorer in place.

A direct link to the GitHub Pages URL is worth keeping somewhere on the Site
too, since some readers prefer the full-screen experience.

## How the JSON files are shaped

`meta.json` — summary numbers used in the header and About tab.

`topics.json` — array of 72 objects, each with `topic_no`, `label`, `cluster`,
`prop`, `rank_in_cluster`, `prob_words[]`, `frex_words[]`,
`longitudinal[{year, prop}]`, `journal_means[{journal, mean_prop}]`.

`clusters.json` — array of 8 objects: `name`, `prop`, `topic_nos[]`,
`longitudinal[]`, `journal_means[]`.

`papers.json` — array of 1,674 objects: `id`, `title`, `year`, `journal`,
`doi`, `url`, `top_topics[{topic_no, prop}]`.

`topic_papers.json` — object keyed by `topic_no` (string `"1"`..`"72"`); each
value is an array of papers with `paper_id`, `title`, `year`, `journal`, `url`,
`prop` (top 25 per topic).

`bridges.json` — array of 2,556 objects: `rank_weak_to_strong`,
`rank_strong_to_weak`, `n_total`, `tier` (`"underdeveloped"` /
`"overdeveloped"` / `"as expected"`), and the topic-pair fields plus
`observed`, `sim_mean`, `ci_lo`, `ci_hi`, `z`.

## Customizing

The whole app is in `index.html`. The CSS variables at the top of the `<style>`
block control colors (including the eight cluster swatches; change the `--c-*`
values to match your manuscript palette). The JavaScript is plain ES6 — no
build step, no node, no bundler.

Three reasonable extension points:

1. Add an "Abstract" column to the Papers tab — pull abstracts in
   `09_Website.R` from CrossRef (you already do this in `02_CrossRef.R`),
   include in `papers.json`, then add the column to `renderPapersTable()` and
   the detail panel.
2. Make Topic detail filterable by year — add a year-range slider above the
   prevalence chart and filter `t.longitudinal` before plotting.
3. Add a download bar — link `data/*.json` files explicitly so methodologists
   can grab the underlying matrices without scraping.
